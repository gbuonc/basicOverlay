Basic Overlay

Basic overlay is a small, developer-friendly jquery plugin focused on showing
overlay/modal windows in web and mobile pages.

It doesn't make any assumption on style and layout, leaving designers and developers maximum creative freedom.

  • Simple and lightweight (3.0kb minified). Works with jQuery 1.7+
  • Designer Friendly: doesn't come with a default style. Just code what you need!
  • Developer Friendly: support remote loaded content via ajax and several event hooks and callbacks
  • Can be customized globally or per call
  • Works on modern and not so modern browsers on web and mobile (Chrome, Firefox, Opera, Safari, IE7 and up)
  • MIT Licensed

Getting Started

  1. Download the script and add the developer or minified version to your page after the jQuery script.

    <script src="http://ajax.googleapis.com/ajax/libs/jquery/1.8/jquery.min.js"></script>
<script src="path/to/jq.basicOverlay.js"></script>

  2. Code and style the content to show in page or in an external file as you normally would (and - usually- hide it via css display:none). No additional css or html is required.

  3. Done!

Basic Usage

The simplest way to open a modal on click is to call the basicOverlay method on a jquery object:

//load the default #modal content by clicking on any element with class openModal
$('.openModal').basicOverlay();

The plugin will look in the current document for a DOM element with id #modal and opens it in the middle of the page with a simple fade in effect. The content is usually hidden and styled via css by the developer/designer. No basic style is provided, so the look and feel can be customized without overwriting unused css selectors.

Modals can be closed by clicking outside the modal window, by pressing ESC key or by adding an element with class .overlay-close inside you modal content. Additional customization is available via configuration object.

The content to be shown in the modal window can be specified in various ways: via element attributes href or data-overlay-modal or passing a configuration object as argument of the function with property content.

Additionally, it is possible to load external content asynchronously via a jquery ajax request. Just specify a remote page (on the same domain, due to browser security restrictions for cross-domain asynchronously requests) and optionally the id of the DOM element to get. The external page can be set via href attribute on a link or via configuration object with property externalPage.

The overlay preserves all styles and scripts found in the target page, so it's possible to show forms, carousels etc. without additional configuration other than that found in the remote document.

  • Local
  • Remote
// html
<button id="btn1">Click!</button>
<a href="#myCustomModal" class="customLink">Open Me</a>
<a href="#" data-overlay-modal="#anotherModal" class="customLink">Open Me</a>
<a href="#myModal" class="customConfig">Open Me</a>
// ...

// javascript
// open the default content #modal
$('#btn1').basicOverlay();
// open #myCustomModal (via href) on the first link, #anotherModal (via data-overlay) on the second one
$('.customLink').basicOverlay();
// open the content specified in config object (overrides html configuration)
$('.customConfig').basicOverlay({content:'#custom'});

// html
<a href="external.html" class="ext">Click</a> 
<a href="external.html" class="ext" data-overlay-modal="#newModal">Click</a> 
<a href="external.html" class="ext2" data-overlay-modal="#notShown">Click</a> 
// ...

// javascript
// open the default content #modal found in external.html on the first link, 
// #newModal (via data-overlay) on the second one
$('.ext').basicOverlay(); 
// open #showMe found in page2.html, as specified in config object 
// overriding href and data-overlay html configuration
$('.ext2').basicOverlay({
   content:'#showMe',
   externalPage: 'page2.html'
});

Configuration

The overlay can be easily styled and configured via a configuration object, that can be passed to a single instance, or globally for all instances in a page via basicOverlay.config property.

// default configuration object
{  
   content:'#modal', 
   externalPage:'', 
   overlayId: 'basicOverlay', 
   overlayColor: '#000000', 
   overlayOpacity: 0.5,    
   closable: true,   
   before: null,
   after:null,
   onClose:null,
   afterClose: null
};

// override global configuration
basicOverlay.config={
   overlayColor: '#fff', 
   content: '#myCustomContent'
}
Property Type Default Value Arguments Description
content STRING '#modal' DOM element to open in overlay as a CSS Selector
externalPage STRING '' URL to an external page on the same domain. If found, it loads DOM element specified by content.
overlayId STRING 'basicOverlay' Id of the global container added to the page by the plugin. Useful for additional styling.
overlayColor STRING '#000000' Overlay background color in hexadecimal value
overlayOpacity NUMBER 0.5 Opacity value of overlay background (0 => full transparent - 1 => full opaque)
closable BOOLEAN true Set true to close the modal by clicking on overlay, pressing ESC key or via close button. By setting this to false the modal can only be closed via basicOverlay.close() method, or by providing an .overlay-close button.
before FUNCTION null element,
callback		 Function to execute before showing the modal window. Useful for additional styling, animations or configuration. Exposes two argument, the jQuery object of the modal content and a callback function (REQUIRED) that needs to be invoked to complete the opening animation
after FUNCTION null element Function to execute after showing the modal window. Useful for DOM manipulation or script initialization. Exposes the jQuery object of the modal content
onClose FUNCTION null element,
callback		 Callback to execute just before the modal window being closed. Useful for closing animations. Exposes two argument, the jQuery object of the modal content and a callback function (REQUIRED) that needs to be invoked to complete the closing animation
afterClose FUNCTION null Callback to execute after the modal window disappears and the added elements are removed from page

Advanced Usage

Is it possible to interact with modals progammaticaly via a standalone basicOverlay function object. This object has two methods, basicOverlay.open() and basicOverlay.close() to open and close the current modal content.

The open() method accepts a configuration object.

// programmatically open a modal overlay
$('#element').click(function(){      
   basicOverlay.open({
      opacity:0,
      closable: false
   });
   return false;
});
// programmatically close the current overlay    
$('#whatever-button').on('click', function(){
   basicOverlay.close();
   return false;
});
// open modal on DOM Loaded
$(function(){
   basicOverlay.open();
});

Three callback hooks (before, after, onClose) return a reference to the current modal content element, so that it can be easily manipulated.

Additionally, before and onClose return a reference to the opening and closing function respectively, to be called to complete the opening and closing process (this is particulary useful to be used as a callback function after a custom animation ends).

basicOverlay.open({
   before: function(el, continue){
      var pos = el.position();  
      // initialize some custom opening animation   
      el.css({'top':(pos.top + 50)+'px', 'opacity':0});    
      // call the 2nd argument as a callback at the and of animation
      el.animate({'top':pos.top, 'opacity': 1}, continue);
      },   
   after: function(el){  
      // do something when the animation is fully completed... 
   },
   onClose: function(el, doClose){
      // set up a closing animation
      var pos = el.position();
      // call the 2nd argument as a callback at the and of animation
      el.animate({'top':(pos.top + 150)+'px', 'opacity': 0}, doClose);
   },
   afterClose: function(){
      // the modal doesn't exist anymore, so there's no reference nor callback
      alert('I\'m done. Thank you!');
   }   	   		      
});
